## 2026-04-05 补充迭代

- [x] 聚合分组 fallback bug 修复与重试规则增强
  - 背景：线上观察到聚合分组首个真实组 `401` 后未切换到下一个真实组，实际 retry 可能仍回到原真实组
  - 修复内容：
    - 新增 `ContextKeyAggregateRetryIndex`
    - 上游返回可重试失败后，下一次 retry 从下一个真实组继续
    - 切到新真实组后，从该真实组自己的最高优先级重新选路
    - 新增聚合分组级 `retry_status_codes` 配置
    - 新增可读 fallback 运行时日志
  - 对应测试：
    - `go test ./service ./controller ./model ./setting/operation_setting`
    - `cd web && bun run build`
  - 结果：通过

- [x] 聚合分组与 channel affinity 冲突修复
  - 背景：命中 `channel affinity`（如内置 `claude cli trace`）且 `skip_retry_on_failure=true` 时，会提前短路 retry，导致聚合分组无法继续真实组 fallback
  - 修复内容：
    - 普通分组保持原有 affinity 行为不变
    - 仅当当前请求为聚合分组时，忽略 `channel affinity` 的 `skip_retry_on_failure`
    - 这样聚合分组仍可在真实组链路内继续高可用切换
  - 对应测试：
    - `go test ./service ./controller ./model ./setting/operation_setting`
  - 结果：通过

- [x] 聚合分组完整交叉测试
